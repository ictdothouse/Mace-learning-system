1. Pengenalan & Deskripsi

Modul Mace merupakan sebuah Platform Pengurusan Pembelajaran (LMS) khas yang direka bentuk untuk memperkasa dan mempermudah ekosistem pembelajaran digital bagi atlet Majlis Sukan Negara (MSN).

Sistem ini dibina bagi menyokong ribuan pengguna secara serentak, membolehkan para atlet menonton video modul pembelajaran secara berstruktur, memuat turun bahan rujukan PDF, menduduki kuiz interaktif, serta menjana sijil penyertaan rasmi secara automatik setelah berjaya menamatkan pengajian modul yang ditetapkan.

2. Seni Bina Sistem (Architecture)

Sistem MACE menggunakan corak seni bina hibrid: Monolithic Backend dengan React SPA Frontend. Ini memastikan kelajuan rendering optimum untuk aplikasi atlet, manakala modul kawalan pentadbir diurus secara server-side rendering (EJS) bagi keselamatan maksimum.

Rajah Aliran Reka Bentuk Seni Bina Sistem

Antaramuka atlet pantas & dinamik. Data diperoleh menerusi REST API secara selamat.

Pelayan Utama

Menguruskan router API, kawalan keselamatan sesi & rendering fail admin (EJS).

Penyimpanan JSON fleksibel dan fail media video/PDF selamat melalui pemprosesan fail R2.

3. Tindanan Teknologi (Technology Stack)

Teknologi yang dipilih berlandaskan kecekapan pembangunan, kestabilan pelayan jangka panjang, dan kelancaran proses naik taraf sistem secara berkala.

FRONTEND (ATLET)
  • React 18 Perpustakaan UI Berkomponen pantas.
  • Vite Alat binaan pembangunan yang pantas.
  • Tailwind Kerangka penggayaan reka bentuk responsif.
  • i18next Sokongan Dwi-Bahasa (BM & EN) tanpa refresh.
BACKEND (EXPRESS)
  • Node/Express Pelayan backend berasaskan event-driven.
  • EJS Templates Sedia-paparan (SSR) untuk Panel Admin.
  • Mongoose Peta skema pengesahan dokumen MongoDB.
  • AWS SDK S3 Sistem Storan R2 (Pemprosesan Media).
INFRASTRUKTUR
  • Ubuntu LTS Ubuntu 22.04 / 24.04 OS pelayan stabil.
  • Nginx Reverse proxy & pelayan fail statik.
  • PM2 Daemon Pemantauan jangka hayat proses backend.
  • Certbot SSL Sijil keselamatan Let's Encrypt percuma.

4. Struktur Direktori Projek

Struktur fail projek diatur secara modular bagi memudahkan pengasingan kod perisian klien (React) dan pelayan (Express).

Format Struktur Folder MACE
/client

Bahagian antaramuka pengguna (Atlet) menggunakan React. Mengandungi sub-folder /components, /pages, dan /locales.

/models

Fail-fail skema pangkalan data MongoDB (Mongoose) seperti: User.js, Athlete.js, Module.js, dan Lesson.js.

/routes

Pengendali laluan API. Mengandungi /admin.js (sesi panel admin EJS), /api.js (data awam React), dan /auth.js (keselamatan login).

/views

Fail reka bentuk templat EJS untuk modul kawalan Admin. Dikawal sepenuhnya oleh backend.

/uploads

Storan cakera pelayan sandaran untuk fail-fail pembelajaran sekiranya storan awan dinyahaktifkan.

app.js

Fail pengisytiharan permulaan pelayan utama (Express Server Entrypoint).

Mongoose Model

6. Ciri-Ciri Utama Sistem

Sistem MACE direka bagi memberikan pengalaman pengguna yang berkualiti tinggi di samping menyediakan alat kawalan data yang stabil untuk pentadbir.

Dwi-Antaramuka (Dual Interface)

Sistem menyokong pengalaman dwi-keperluan; atlet berinteraksi dengan SPA React yang dinamik, pantas dan moden manakala pentadbir mengurus sistem menggunakan laman Server-Side Rendered (EJS) yang ringkas, kukuh, dan sangat selamat.

Dwi-Bahasa (Localization)

Sistem menyokong pertukaran bahasa serta-merta tanpa perlu memuat semula halaman menggunakan fail terjemahan i18next di bahagian frontend untuk Bahasa Melayu (BM) dan English (EN).

Migrasi Media 1-Klik (Onload / Offload)

Fleksibiliti storan membolehkan pentadbir membuat keputusan storan: memuat naik fail video dan PDF ke Sistem Storan R2 (Pemprosesan Fail Media) atau mengekalkannya di storan cakera keras tempatan (Local Storage /uploads) dalam satu butang klik sahaja.

Anti-Skip Tontonan Video

Sistem merekodkan masa sebenar tontonan. Atlet dilarang melangkau (skip) bahagian hadapan video bagi memastikan kelayakan perolehan ilmu dan sijil yang tulen.

Penjanaan Sijil Automatik (Auto-Sijil PDF)

Setelah atlet berjaya mencapai tahap peratusan markah atau tontonan modul yang ditetapkan, sistem memanggil koordinat templat sijil dan membina satu fail PDF sijil digital berpematuhan standard yang sedia untuk dimuat turun.

7. Dokumentasi Laluan API (Endpoints)

Laluan API utama dipecahkan mengikut tahap keselamatan dan pengguna sasaran bagi memastikan kawalan capaian data terbaik.

Kategori Laluan Format Laluan / Prefix Jenis Keselamatan Fungsi / Deskripsi
Pentadbir (Admin) /admin-mace/* Sesi Tempatan Admin Pemasukan modul pembelajaran, pemantauan progress atlet, konfigurasi sijil dan penjenamaan (branding).
Pengepala Autentikasi /api/auth/* Terbuka (Bebas) Pengesahan No. Kad Pengenalan untuk menjana token akses JWT atlet.
Akses Awam /api/* Bebas Akses Klien Menarik konfigurasi logo, nama sistem, skema warna untuk paparan pertama React.
Akses Terlindung /api/athlete/* Bearer Token (JWT) Mengambil profil peribadi atlet, kemajuan pelajaran, keputusan kuiz, dan status muat turun sijil.

8. Persekitaran Pelayan (Server Environment)

Sistem MACE direka untuk dihoskan (deployed) pada pelayan Virtual Private Server (VPS) berasaskan Linux (disyorkan Ubuntu 22.04 / 24.04 LTS) untuk memastikan kestabilan dan prestasi optimum.

Pakej Kebergantungan Teknologi (Tech Stack Dependencies)

Pemasangan sistem pada peringkat pelayan akan mengkonfigurasi keseluruhan kitaran perkhidmatan aplikasi dengan susunan perisian berikut:

  • Runtime Perisian: Node.js LTS untuk memproses logik aplikasi backend dan frontend berasaskan JavaScript.
  • Pangkalan Data: MongoDB Community Server untuk pengurusan data dokumen (NoSQL) berskala besar secara lokal.
  • Reverse Proxy: Nginx Web Server untuk mengawal trafik luar dan memautkannya kepada port aplikasi dalaman.
  • Pengurusan Proses: Daemon PM2 digunakan untuk memastikan aplikasi Node.js sentiasa hidup (24/7) dan dimulakan semula (auto-restart) jika ralat berlaku.
  • Keselamatan Rangkaian: Sokongan sambungan SSL (HTTPS) dan kawalan IP dalaman melalui konfigurasi Firewall (UFW).
