Panduan Pemasangan Automatik (Easy Install) - Sistem MACE

Panduan ini akan membimbing anda memasang keseluruhan sistem MACE ke dalam pelayan (server) VPS baru menggunakan skrip automatik. Skrip ini akan memasang MongoDB, Node.js, NPM, Nginx, dan PM2 secara serentak.

Keperluan Pelayan Tinggi (Khas untuk kapasiti 8,000+ Atlet):
1. OS: Ubuntu 22.04 LTS atau Ubuntu 24.04 LTS (Disyorkan).
2. RAM: 8GB hingga 16GB (Wajib untuk menampung pangkalan data MongoDB & trafik ribuan pelajar serentak).
3. CPU: 4 Cores hingga 8 Cores.
4. Storan (Disk Space): Minimum 100GB ke 200GB NVMe SSD (Wajib, kerana semua video & imej kini disimpan secara tempatan dan SSD NVMe amat pantas untuk penstriman video).
5. Rangkaian (Network): Kelajuan sekurang-kurangnya 1 Gbps dengan kuota data (Bandwidth) yang tinggi.
6. Port / Firewall: Sila pastikan Port 80 (HTTP), 443 (HTTPS), 22 (SSH), dan 4000 (Custom TCP - MonitorPanel) wajib dibuka pada ruangan firewall / Security Group pelayan.
7. Akses: Menyediakan akses sebagai pengguna root (atau sudo) untuk tujuan *deployment*.

Sila maklumkan 3 perkara INI kepada pihak IT (Sangat Penting):
8. Pelayan Kosong (Fresh OS): Pelayan Ubuntu mestilah *kosong*. JANGAN install web panel seperti cPanel, Plesk, atau Apache. Skrip kami akan memasang Nginx dan ia memerlukan Port 80/443 bebas (unoccupied).
9. Domain DNS / Alamat IP: Jika nama domain (cth: mace.msn.gov.my) sudah bersedia, pastikan ia telah disambungkan ke alamat IP pelayan (A Record) sebelum mula. Namun, jika domain belum siap, pihak IT boleh teruskan pemasangan menggunakan Alamat IP Pelayan sahaja terlebih dahulu. Sistem tetap akan berfungsi (tanpa SSL/HTTPS). HTTPS boleh dipasang di kemudian hari apabila domain rasmi telah diluluskan.
10. Akses Internet Keluar (Outbound Network): Pelayan mesti mempunyai kebenaran memuat turun terus dari Internet (unrestricted outbound access) untuk memuat turun pakej Node.js, MongoDB, GitHub dan NPM.
11. Pemetaan IP (Public & Private IP NAT): Memandangkan pelayan berada di dalam pusat data kerajaan, ia mungkin diletakkan di belakang Hardware Firewall jabatan. Pastikan pihak IT memetakan (Port Forward) Public IP dengan betul kepada Private IP (Local IP) pelayan Ubuntu tersebut supaya ia boleh diakses dari luar. Trafik masuk untuk port 80, 443, dan 4000 mesti dihalakan terus ke Private IP mesin ini.

Langkah 1: Log Masuk ke Pelayan (SSH)

Gunakan Terminal (Mac) atau Command Prompt / PowerShell (Windows) untuk log masuk ke VPS anda.

ssh root@ALAMAT_IP_SERVER_ANDA

* Masukkan kata laluan pelayan anda apabila diminta.

Langkah 2: Jalankan Skrip Pemasangan Automatik (1-Click Install)

Anda hanya perlu menyalin (copy) dan menampal (paste) arahan tunggal ini ke dalam terminal anda. Skrip pintar ini akan memuat turun keseluruhan sistem secara automatik dari GitHub dan memasangnya untuk anda.

wget -qO- https://raw.githubusercontent.com/ictdothouse/Mace-learning-system/main/install.sh | sudo bash

Skrip ini akan mengambil masa 2-5 minit untuk memuat turun semua perisian yang diperlukan (NodeJS 20, MongoDB 7, dll).

SEMASA PEMASANGAN: Skrip akan bertanya soalan interaktif kepada anda. Pastikan anda mempunyai maklumat ini siap sedia:
  • Kekunci Cloudflare R2 (Access Key & Secret Key) - Boleh guna yang sedia ada.
  • Kata laluan yang anda mahukan untuk akaun Admin baru.

Langkah 3: Hidupkan Sistem Menggunakan PM2

Setelah selesai, skrip telah membina fail .env secara automatik dan menyusun front-end React. Kini, anda hanya perlu hidupkan aplikasi untuk berjalan 24/7 di latar belakang. Mula-mula, masuk ke folder sistem:

cd Mace-learning-system pm2 start app.js --name "mace-system" pm2 start monitorpanel.js --name "mace-monitor"

Untuk memastikan aplikasi hidup semula jika pelayan dihidupkan semula (reboot):

pm2 startup pm2 save

Langkah 4: Uji Laman Web Anda

Sistem anda kini sedang berjalan di pelayan tempatan pada port 3000 (bersama dengan Nginx di port 80). Buka pelayar web (browser) komputer anda dan layari alamat IP pelayan.

http://ALAMAT_IP_SERVER_ANDA

Pastikan anda menaip http:// (tiada 's'). Sesetengah pelayar web akan menukarnya kepada https:// secara automatik yang menyebabkan ralat.

Langkah 5 (PENTING): Migrasi Data Lama

Sistem ini sekarang ibarat 'kereta baru tanpa penumpang'. Pangkalan data MongoDB di dalam VPS adalah KOSONG. Anda perlu memuat naik fail JSON yang dimuat turun dari pelayan lama.

  1. Buka http://ALAMAT_IP_SERVER_ANDA/admin-mace.
  2. Anda akan diminta untuk memasukkan maklumat Basic Auth Master Admin. Masukkan:
    • Username: admin
    • Password: Kata laluan yang anda masukkan semasa menjalankan skrip install.sh (atau admin123 secara lalai/default).
  3. Selepas berjaya log masuk, pergi ke menu Settings -> Database Migration.
  4. Di bawah bahagian Muat Naik / Import (Restore), pilih fail mace_system_data.json yang anda telah eksport dari pelayan lama.
  5. Klik "Import". Pangkalan data anda akan dipulihkan serta merta!

Langkah 6: Akses MonitorPanel

Bagi memudahkan penyelenggaraan, sistem ini dilengkapi dengan MonitorPanel terbina dalam. Daripada menggunakan terminal Linux untuk restart atau mengemaskini (redeploy) kod baru, anda boleh lakukannya terus dari antaramuka web.

  1. Buka http://ALAMAT_IP_SERVER_ANDA:4000 (Pastikan bermula dengan http:// dan diakhiri dengan :4000).
  2. Log masuk menggunakan kata laluan Master Admin yang sama.
  3. Anda boleh melihat status RAM/CPU, menyemak log ralat, dan menekan butang Restart Sistem atau Git Pull.

Nota Penting: Anda DIWAJIBKAN untuk membuka (Allow) Port 4000 (Custom TCP) di dalam panel Firewall / Security Group pelayan anda terlebih dahulu. Jika tidak, MonitorPanel ini langsung tidak akan dapat diakses dari luar.

Penyelesaian Masalah 1: Ralat 502 Bad Gateway

Jika anda melayari alamat IP anda dan mendapat ralat 502 Bad Gateway, ia berkemungkinan besar berpunca daripada Pangkalan Data (MongoDB) yang terhenti atau terpadam. Sistem tidak dapat berhubung dengan database lalu menyebabkan ralat ini.

Jalankan arahan berikut di terminal VPS untuk menghidupkan semula database dan sistem MACE:

# 1. Hidupkan semula MongoDB sudo systemctl restart mongod # 2. Periksa statusnya (pastikan ia 'active') sudo systemctl status mongod # (Tekan butang 'q' untuk keluar dari log status) # 3. Mulakan semula sistem MACE sudo pm2 restart mace-system

Penyelesaian Masalah 2: PM2 Tersekat (Hang/Stuck)

Jika arahan PM2 seperti pm2 restart tidak berfungsi dan skrin terminal anda tersekat, ia bermakna proses di belakang tabir telah terhenti. Gunakan pelan tindakan ini untuk mematikan dan menghidupkannya secara paksa:

# Tekan Ctrl + C di keyboard anda untuk keluar dari arahan yang tersekat # 1. Matikan enjin PM2 secara paksa sudo pkill -9 -f pm2 # 2. Matikan apa-apa proses Node.js yang tersangkut (pilihan) sudo pkill -9 -f node # 3. Hidupkan semula PM2, Sistem MACE dan MonitorPanel sudo pm2 start app.js --name "mace-system" sudo pm2 start monitorpanel.js --name "mace-monitor"

Info Tambahan: Cara Mengemaskini (Redeploy) Sistem

Jika terdapat kod baru yang telah di-push ke GitHub dan anda ingin mengemas kini sistem MACE di pelayan anda, jalankan arahan berikut di terminal VPS:

cd ~/Mace-learning-system git pull origin main # (Pilihan) Jika ada perubahan pada antaramuka (Frontend/React): # cd client && npm install && npm run build && cd .. sudo pm2 restart mace-system
